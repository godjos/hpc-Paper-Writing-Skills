# Technical Challenge Version 3: Novel Workload

1. State the new workload or deployment setting.
2. Decompose the challenge into major barriers.
3. Explain why each barrier is hard.
4. Use the decomposition to motivate the method.

Terminology anchors:

1. Assign each barrier a stable name, such as communication amplification, synchronization delay, memory pressure, data-placement mismatch, or runtime scheduling overhead.
2. For novel workloads, define new terms before using abbreviations.
3. Tie each barrier to a metric, platform condition, or workload property.
