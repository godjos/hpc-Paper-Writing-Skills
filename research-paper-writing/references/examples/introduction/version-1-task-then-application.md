# Introduction Version 1: Workload First

1. Define the workload or system in one sentence.
2. Explain why it matters for HPC practice.
3. Mention 2-3 representative applications.
4. Add the target metric or constraint.

Terminology anchors:

1. Name the workload and platform exactly, for example `distributed sparse matrix multiplication on GPU clusters`.
2. Name the execution entity and metric, for example `MPI ranks`, `GPU kernels`, `end-to-end runtime`, or `parallel efficiency`.
3. Avoid generic replacements such as `task`, `resource`, or `performance` when a precise HPC term is available.
