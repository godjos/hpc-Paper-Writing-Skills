# Pipeline Version 1: One Contribution, Multiple Advantages

1. Name the main system idea.
2. State the main contribution.
3. List 2-3 measurable advantages.
4. Connect them to the bottleneck.

Terminology anchors:

1. Name the contribution with a system term, such as communication-computation overlap, locality-aware scheduling, kernel fusion, or topology-aware placement.
2. Express advantages as metrics: runtime, throughput, memory bandwidth, parallel efficiency, energy, or tail latency.
