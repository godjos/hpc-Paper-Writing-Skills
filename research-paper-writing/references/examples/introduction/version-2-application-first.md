# Introduction Version 2: Application First

1. Open with a practical HPC application.
2. State why the application is demanding.
3. Add the specific target metric or bottleneck.

Terminology anchors:

1. Use the exact application workload, such as weather simulation, molecular dynamics, large-scale graph processing, or distributed training.
2. State the demanding property as a measurable HPC requirement: strong scaling, weak scaling, memory bandwidth, I/O bandwidth, latency, or throughput.
